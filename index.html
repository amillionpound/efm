快速开始

1 检查环境要求

Windows 7及以上版本（32位操作系统），确保服务器可访问外网或测试网段。

2 安装基础环境

安装 JDK 1.8 + Tomcat 8.5 + U盾驱动程序 + 数字证书。

3 切换生产模式

修改 setting 目录下的配置文件，将 ENV=test 切换为 ENV=prd。

4 验证连通性

通过 telnet 和本地 URL 测试联通，发送查询报文验证U盾签名。

前置机部署手册

当前版本 v1.7（2024-04-28）

设备要求

Windows 7及以上版本，32位操作系统。

1 安装 JDK

如已安装 JDK 1.8 版本可跳过此步。

运行前置安装包中的 jdk-8u31-windows-x64.exe，安装路径选择默认路径，一路点击下一步直至完成。

安装成功后，打开 cmd 验证 JDK 版本：

java -version
配置 JDK 环境变量（默认路径）：
JAVA_HOME = C:\Program Files\Java\jdk1.8.0_31
CLASS_PATH = .;%JAVA_HOME%\lib;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar
注意：需将 jni 控件签名文件 MdsmJni.dll 放入路径
C:\Program Files\Java\jdk1.8.0_31\jre\bin\ 下。如已安装其他版本 JDK，放到对应 .\jre\bin 目录下。控件有32位和64位版本，JDK需对应，与操作系统无关。
2 安装 Tomcat

如已安装 Tomcat 可跳过此步。

解压安装包中的 apache-tomcat-8.5.100 到 D:\GHBANKENTFIN 目录（只用 apache-tomcat-8.5.100 文件夹）。

双击运行 ssmd 安装程序（签名程序），安装时默认下一步即可。

系统架构判断：
查看"系统信息" → 系统类型，显示"x86"则系统为32位（386）；显示"x64"则系统为64位（amd64）。

启动 Tomcat（双击）：

D:\GHBANKENTFIN\apache-tomcat-8.5.100\bin\startup.bat

启动成功后出现命令行窗口，不要关闭该窗口。检查是否部署了应用 ghbankgayway4ent.war。

3 安装U盾驱动
💡 测试环境无需U盾，本步骤仅适用于生产环境。测试阶段可直接跳过步骤3、4、7，进入步骤5配置环境。

（生产环境）运行安装包中的 UKey管理工具20210112.exe，完成UKey管理工具安装。

插入U盾后，验证证书是否在系统中：

开始 → 运行 → certmgr.msc → 查看「GDHXBANK\证书」目录下是否有证书
4 修改U盾密码
💡 测试环境无需U盾，本步骤仅适用于生产环境。

（生产环境）首次使用应自定义修改U盾密码（初始密码：123456），如未修改，推荐通过"UKey管理工具"修改。

提示需要插入U盾，请确认后修改。
5 切换环境模式

找到配置文件目录：

D:\GHBANKENTFIN\apache-tomcat-8.5.100\webapps\ghbankgayway4ent\WEB-INF\classes\config\setting\

文件1：gayway.properties

将 ENV=test 修改为 ENV=prd 即可切换为生产环境

文件2：gayway.prd.properties

将 custNo=5600057376 修改为 custNo=【客户实际客户号】
6 验证连通性

测试环境：

telnet 183.63.138.12 8803 telnet 183.63.138.117 8803

使用浏览器访问以下URL，返回登录提示即表示畅通：

http://localhost:8080/ghbankgayway4ent/rest/gayway

生产环境：

telnet open.ghbank.com.cn 443

同样访问：

http://localhost:8080/ghbankgayway4ent/rest/gayway
7 前置机U盾证书验证
💡 测试环境无需U盾，测试阶段使用 SKEY 模式（certType=SKEY），不会弹出U盾密码输入框，可直接发送报文测试。本步骤仅适用于生产环境。

（生产环境）首次发送报文时，需在部署前置机的服务器上输入U盾密码。推荐首次发送查询报文进行测试，如账户余额查询（接口编号：ZTSA63Q01）。

在服务器上输入U盾密码后，系统会缓存密码，后续不再弹窗要求输入。如更换U盾需重新输入密码。
8 前置应用启停

启动：双击运行

D:\GHBANKENTFIN\apache-tomcat-8.5.100\bin\startup.bat

停止：直接关闭命令行窗口即可。

如需设置定时自动重启（推荐每晚重启保证运行稳定），可参考以下设置：

定时任务设置方法：
1. WIN+R → taskschd.msc 打开任务计划程序
2. 创建定时关闭任务：每日执行 D:\GHBANKENTFIN\apache-tomcat-8.5.100\bin\timeshutdown.bat
3. 创建定时启动任务：在关闭任务5-10分钟后，执行 D:\GHBANKENTFIN\apache-tomcat-8.5.100\bin\timestart.bat
4. 选"一次性任务"，点击运行验证

接口文档

接入标准 v6.0 | 通讯协议 XML/HTTPS | 接口编号规则：ZTSA63 + 类型(Q查询/F动账/M复合)

通讯架构

银行服务器：

测试：POST http://183.63.138.12:8803/api/custodian/bankEnterpriseDirectInterface/baseInterface/V1 生产：POST https://open.ghbank.com.cn/api/custodian/bankEnterpriseDirectInterface/baseInterface/V1

前置机（企业本地）：

POST http://[前置机IP]:8080/ghbankgayway4ent/rest/gayway

HTTP头要求：Content-Length、Content-Type: application/xml; charset=UTF-8

通讯方式：企业ERP与前置机间用HTTP；前置机与银行间用HTTPS双向认证

报文格式：XML标准报文，UTF-8编码，ROOT为根元素

安全规范要点
  • 传输加密：前置机与银行间使用HTTPS协议，保证通道加密
  • 数字签名：使用U盾内企业数字证书进行签名，确保报文的完整性和不可否认性
  • 报文防篡改：签名为ROOT元素的BASE64编码后进行算法签名，结果放入SIGNATURE标签
  • 防重放：涉及资金变动的交易报文，流水号必须保证唯一性，同一流水号只允许执行一次
  • 验签：返回报文去SIGNATURE元素后，用SHA1算法取ROOT摘要，再用SHA1RSA验签
  • 测试环境：支持SKEY免U盾模式（仅在测试阶段），配置certType=SKEY

接入指引

企业客户接入银企直联系统的完整流程

接入协调人

由我行科技部技术同事协调企业搭建系统的程序员。通过企业微信/微信对接群沟通，群内答疑。测试完成后确认无误，通过对接群通知切换到生产环境。

一、接入前须知
  • 先在测试环境进行连通性测试
  • 测试环境会提供必要的测试账号以及测试网段（如银行内部网络）
  • 测试数据无固定格式（转账信息、存款信息等），在微信群内发送即可
二、测试环境

🟢 测试环境无需U盾，使用 SKEY 免U盾模式进行测试，部署手册中的步骤3（安装U盾驱动）、步骤4（修改U盾密码）、步骤7（U盾证书验证）可直接跳过。

测试环境参数：参见《部署手册》中 1.7节 切换环境模式 下的测试环境配置（ENV=test，certType=SKEY）。

⚠️ 测试环境IP地址可能不是最新的，如无法连接请咨询对接群。可以先发一笔转账，从返回报文获取测试环境实际地址，或参考 1.9节 常见问题排查。

三、正式接入
  • 签署协议：需签署《银企直联协议》《服务申请表》《企业证书授权书》等文件
  • 客户号：即"企业号"，指与我行签约的企业客户号，联系您的客户经理获取
  • 正式UK：完成签约后发放正式环境的UKEY
  • 切换配置：将 ENV=test 改为 ENV=prd，客户号改为实际客户号
  • 验证：通过查询报文（ZTSA63Q01）验证正式环境连通

启动Tomcat报错：Can't load IA 32-bit .dll on AMD 64-bit platform ▼

原因：JDK版本不匹配或环境变量配置错误。

排查步骤：

  1. 检查 JAVA_HOME 环境变量是否正确
  2. 在 cmd 中执行 java -version 确认版本
  3. 找到并确认对应的JDK安装目录（例如 jdk1.8.0_111）
  4. 另一种可能是绿色版JDK，需修改加载的库文件
Tomcat启动后浏览器访问404/无法访问 ▼

排查步骤：

  1. 防火墙未开放端口
  2. 环境变量配置有多个JRE冲突
  3. jni控件文件（MdsmJni.dll）未放置到正确位置
  4. 端口被占用：netstat -ano | findstr 8080
  5. 如需修改端口：编辑 apache-tomcat-8.5.100\conf\server.xml
  6. 注意：修改端口后如更换服务器，需同时修改防火墙
签名一直失败 / 验签不通过 ▼

常见原因：

  1. 首次发送报文时Tomcat会弹U盾密码输入框——如果弹窗出现时没有输入密码就关闭了，会导致持续签名失败
  2. U盾未正确插入或驱动未安装
  3. 证书未导入（用 certmgr.msc 检查是否有 GDHXBANK 目录的证书）
  4. key库配置问题
证书检查：找不到GDHXBANK证书 ▼

在 cmd 中运行 certmgr.msc，搜索 "ghbank"，确认是否有证书。

如果没有证书：

  • 重新导入证书（需插入U盾）
  • 检查U盾驱动程序是否安装成功
使用财务软件或司库系统对接，银行侧有区别吗 ▼

没有区别。企业无论使用财务软件还是司库系统对接银行的银企直联系统，对银行来说都是一样的，处理方式完全相同。

U盾证书有效期快到了 / 刚开通就提示有效期不足 ▼

U盾证书有效期是5年，但可能存在预制备的证书，导致企业开通银企直联后没多久就提示有效期快到了。

解决方法：请联系对应的客户经理，带齐资料以及U盾到我行网点更新证书。

ssmd安装程序弹窗后立即自动关闭 ▼

原因：电脑上安装了抓包工具（如Fiddler、Wireshark等），ssmd检测到后自动关闭。

解决方法：先关闭所有抓包工具，再重新运行ssmd安装程序。
